## 3. PSWiki / Plant Simulation Community forum post

**Subject:** [Open Source] PlantSim-Agent v0.1.0 — a GitHub Copilot agent for SimTalk + `.psfm` analysis

> Hi all,
>
> I've just released v0.1.0 of **PlantSim-Agent**, an open-source GitHub Copilot integration built specifically for Plant Simulation work.
>
> **What it does**
> - Answers questions about your **own licensed PTS Help** (local SQLite FTS5 index — never uploaded)
> - Writes SimTalk 2.0 code with a mandatory **API Evidence Table** so every symbol traces to its Help source
> - Reads `.psfm` projects: `find_method`, `find_callers`, `get_object_graph`, lints SimTalk methods
> - A separate **citation-reviewer** sub-agent regex-checks every reply; missing or hallucinated sources are rejected
>
> **Why I built it**
> Generic LLMs hallucinate SimTalk APIs, fall back to deprecated 1.0 syntax, and can't reason about a specific `.psfm` model. The built-in AI features in PS itself are improving but still conservative. I wanted a workflow where AI helps with the boring 80 % (lookups, boilerplate, refactor candidates) while staying honest about what it doesn't know.
>
> **Status**
> - 7 MCP tools, 3 skills, 2 agents
> - 122 unit tests + 30/30 evaluation set passing
> - Tested on a large-scale logistics-automation `.psfm` model (25,463 objects / 7,698 flow edges)
> - **Windows only** (PS is Windows-only)
> - Requires VS Code 1.99+, GitHub Copilot, Python 3.10+
>
> **Try it**
> ```
> git clone https://github.com/JackySummerfield/PlantSim-Agent.git
> cd PlantSim-Agent
> .\scripts\install.ps1
> pip install -e mcp/
> plantsim-copilot-mcp init
> ```
> Full docs + cold-install checklist in the repo.
>
> **What's next (v0.2)**
> Vector retrieval (FTS5 has known recall gaps), real SimTalk lexer/AST for `validate_simtalk`, and read-write `.psfm` patching with the same mandatory citation contract.
>
> Issues / discussions / PRs welcome — especially from anyone doing AGV, conveyor, or MES integration. License is MIT.
>
