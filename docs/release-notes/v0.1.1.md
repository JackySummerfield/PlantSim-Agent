# v0.1.1 — One-command MCP server registration

Quality-of-life patch release. Cuts the cold-install path from "find
your OS's `mcp.json`, parse JSON, merge a server entry by hand without
clobbering anything else, find the right `command` value" down to one
command — automatically called by `install.ps1`.

No breaking changes. The manual JSON path is still supported and now
lives in [`docs/manual-mcp.md`](../manual-mcp.md).

## What's new

### `plantsim-copilot-mcp register-vscode`

A new subcommand that does the right thing without you having to look
anything up:

```powershell
plantsim-copilot-mcp register-vscode
```

It will:

- Locate VS Code's user-level `mcp.json` for your OS
  (Windows `%APPDATA%\Code\User`, macOS
  `~/Library/Application Support/Code/User`, Linux `~/.config/Code/User`)
- Merge the `plantsim-copilot-mcp` server entry idempotently — your
  other servers, inputs, and config blocks are preserved verbatim
- Back up the previous file as `mcp.json.bak-<YYYYMMDD-HHMMSS>` before
  any overwrite
- Refuse (exit code 2) to clobber a differing existing entry under
  the same name unless you pass `--force`

Useful flags: `--target <path>` (override the file), `--command <path>`
(force a specific executable), `--absolute` (write the resolved
absolute path of `plantsim-copilot-mcp` instead of the bare name —
helpful when VS Code launches outside your conda env), `--dry-run`
(print the merged JSON, don't write).

### `install.ps1` calls it for you

The PowerShell installer now runs `register-vscode` at the end of its
symlink setup. If the console-script isn't on `PATH` yet (typical on
the very first run before `pip install -e mcp/`), the installer prints
a non-fatal skip notice and tells you to rerun.

### Simplified quick-start in the README

Step 4 went from a 30-line JSON snippet with platform-specific path
tables to:

```powershell
plantsim-copilot-mcp register-vscode
```

…with a one-line note that `install.ps1` already did it for you.

## Upgrade path

```powershell
cd $HOME/.copilot/plantsim-agent
git pull
.\scripts\install.ps1     # picks up the new register-vscode step
```

Or just reach for the new command directly:

```powershell
plantsim-copilot-mcp register-vscode
```
