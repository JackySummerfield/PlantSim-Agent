# v0.1.0 — First Public Release

> 让 AI 真正成为 Plant Simulation 的副驾驶。

PlantSim-Agent 把 GitHub Copilot 变成一个能查帮助文档、写 SimTalk 代码、读 `.psfm` 项目的「Plant Simulation 副驾驶」——并且每个答案都必须给出来源，不允许凭空捏造。

## What's in the box

### 🤖 7 个 MCP Tool

| Tool | 用途 |
|---|---|
| `search_help` | 全文检索本地 Help 知识库（FTS5） |
| `get_api` | 按名字精确查 SimTalk API，找不到时返回 `did_you_mean` |
| `find_method` | 在 `.psfm` 项目里按名找 Method，并列出继承覆写 |
| `find_callers` | 反向查谁调用了某个 identifier |
| `search_code` | 在所有 SimTalk method body 里做 FTS |
| `get_object_graph` | 返回对象的继承 + 物流邻居 |
| `validate_simtalk` | SimTalk 2.0 lint（ST001-ST004） |

### 🛠 3 个 Skill

- **`plantsim-kb-qa`** — 文档问答级联（`get_api` → `did_you_mean` 重试 → `search_help` → 拒绝），强制 `**Sources:**` 锚点
- **`plantsim-code-author`** — Symbol-Lookup-Cascade + API Evidence Table + Refuse-to-Guess
- **`plantsim-project-analyst`** — 四个子流程（Locate / Trace / Map / Audit），永远开启继承审计

### 🛡 2 个 Agent

- **`plantsim-copilot`** — 用户面 orchestrator，自动按意图路由到对应 skill，每次回答后调度 reviewer
- **`citation-reviewer`** — 单一职责的 sub-agent，三段 regex 检查（anchor 存在 / 来源质量 / 拒绝纪律），返回 JSON 裁决

### 🧙 一键开箱

```powershell
git clone https://github.com/JackySummerfield/PlantSim-Agent.git $HOME/.copilot/plantsim-agent
cd $HOME/.copilot/plantsim-agent
.\scripts\install.ps1
pip install -e mcp/
plantsim-copilot-mcp init           # 交互式向导
# 或 CI 自动化：
plantsim-copilot-mcp init --non-interactive --kb-root .\kb_minimal --build
```

## Highlights

- **本地优先**：所有索引在你电脑上的 SQLite FTS5 里，问答全程不出机器
- **可追溯**：每个回答都附 `**Sources:**` 或 `API Evidence Table`，回链到 Help 章节或代码行
- **拒绝瞎编**：找不到来源就明确说 `❌ Cannot verify`，不让训练数据里的 SimTalk 1.0 老语法溜进去
- **真实项目验证**：已在大型物流自动化仿真模型（25,463 objects / 7,698 flow edges）上跑通；并在真实 SimTalk 方法里查出 4 个 lint 问题

## Quality bar

- **122 个单元测试**（mcp + cold-install）全部通过
- **30/30 评估通过**：20 个 QA 召回 + 10 个 citation-recall fixture（`tests/eval/`，`pytest --run-eval`）
- **冷装回归**：3 个 subprocess 测试守门 `init → build → serve` 全流程，外加 [`docs/cold-install.md`](docs/cold-install.md) 7 步手动 checklist

## Known limitations (→ v0.2)

- FTS5 索引不包含 section 标题，自然语言查询召回受限 → v0.2 上向量检索
- `validate_simtalk` 只有 4 条正则规则 → v0.2 升级到 lexer/AST
- 项目只读，不能写回 → v0.2 加 `apply_patch` / `replace_method_body`，沿用 citation 审计契约

## Compatibility

- **Windows only**（Plant Simulation 本体只在 Windows 上运行；agent 没有独立价值）
- VS Code 1.99+，GitHub Copilot 最新版
- Python 3.10+
- Tecnomatix Plant Simulation 2024 SP3 / 2025 SP1 测试通过；2025 SP4 (PTS Help 25.04) 是 v0.1 默认目标
